# Règles internationalisation (i18n)

## Activation

Conditionnel — activé si Phase 1 Q4 = `Oui`.

Si activé :
- Dossier `resources/i18n/` obligatoire.
- Fichiers source `.ts` + compilés `.qm`.
- Aucune chaîne en dur dans Views, Controllers, Models.

Si désactivé :
- Aucun appel `self.tr()` requis (les chaînes restent en français en dur dans les Views).
- Pas de dossier `resources/i18n/`.

---

## Stack

- `PyQt6.QtCore.QTranslator` (natif Qt, zéro dépendance externe).
- `pylupdate6` (fourni avec PyQt6) pour extraire les chaînes → `.ts`.
- `lrelease` (Qt Linguist Tools — à installer séparément si besoin) pour compiler `.ts` → `.qm`.

---

## Structure obligatoire

```
resources/
└── i18n/
    ├── app_fr.ts             # Source FR (traductions humaines)
    ├── app_en.ts             # Source EN
    ├── app_fr.qm             # Compilé — chargé par l'application
    └── app_en.qm             # Compilé
```

---

## Conventions

- Toute chaîne visible utilisateur passe par `self.tr("texte")` dans les Views.
- Pas de `self.tr` dans Models / Controllers — la logique métier ignore la langue.
- Les exceptions levées par les Models contiennent du français (langue support) ; le
  Controller traduit le message côté View via `self.tr(str(exception))` si nécessaire.
- Format de chaîne : éviter les concaténations. Utiliser `self.tr("Bonjour %1").arg(name)`.

---

## Initialisation dans main.py

```python
from PyQt6.QtCore import QTranslator, QLocale
from PyQt6.QtWidgets import QApplication
from pathlib import Path

import config


def install_translator(app: QApplication) -> QTranslator:
    """Installe le traducteur selon la préférence utilisateur."""
    locale = _load_user_locale()  # depuis preferences.json
    translator = QTranslator()
    qm_path = Path("resources/i18n") / f"app_{locale}.qm"
    if translator.load(str(qm_path)):
        app.installTranslator(translator)
    return translator
```

## Workflow développeur

### 1. Extraire les chaînes depuis le code

```powershell
pylupdate6 (Get-ChildItem -Recurse views\*.py | Select-Object -ExpandProperty FullName) `
           -ts resources\i18n\app_fr.ts resources\i18n\app_en.ts
```

### 2. Traduire (Qt Linguist)

Ouvrir chaque `.ts` dans Qt Linguist, remplir les traductions, sauvegarder.

### 3. Compiler en .qm

```powershell
lrelease resources\i18n\app_fr.ts resources\i18n\app_en.ts
```

`lrelease` n'est pas inclus dans PyQt6. Deux options :
- Installer Qt Linguist Tools séparément (download.qt.io).
- Utiliser `PyQt6.lrelease` si disponible dans la version PyQt6 installée.

### 4. Script d'automatisation — `scripts/i18n_update.ps1`

Livré dans le dernier lot applicatif si i18n activée :

```powershell
# scripts/i18n_update.ps1
$files = Get-ChildItem -Recurse -Include *.py -Path views,controllers `
         | Select-Object -ExpandProperty FullName
pylupdate6 $files -ts resources\i18n\app_fr.ts resources\i18n\app_en.ts
Write-Host "✓ Chaînes extraites — éditer les .ts puis lancer i18n_compile.ps1"
```

Et `scripts/i18n_compile.ps1` :

```powershell
Get-ChildItem resources\i18n\*.ts | ForEach-Object {
    lrelease $_.FullName
}
Write-Host "✓ Fichiers .qm générés"
```

## Changement de langue à chaud — non supporté

- Pas de réinstallation dynamique du `QTranslator` à la volée.
- Changement de langue = écriture dans `preferences.json` + invitation à redémarrer.
- Cette contrainte évite la complexité de retraduction de tous les widgets ouverts.

---

## Interdictions

- Zéro chaîne utilisateur en dur dans Views (`QLabel("Texte")` → `QLabel(self.tr("Texte"))`).
- Zéro chaîne UI dans Models ou Controllers.
- Pas de rechargement à chaud des traductions.

---

## Vérification d'intégrité

1. Si i18n = Oui : `resources/i18n/app_{fr,en}.ts` et `.qm` présents.
2. Aucune chaîne `QLabel("…")`, `QPushButton("…")`, `QMessageBox("…")` en français dur dans `views/`.
3. `QTranslator` installé dans `main.py` avant création de la `MainWindow`.
4. Préférence `langue` lue depuis `preferences.json` au démarrage.
