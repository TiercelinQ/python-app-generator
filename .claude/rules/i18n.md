# Internationalization (i18n) rules

## Activation

Conditional — enabled if Phase 1 Q4 = `Yes`.

If enabled:
- `resources/i18n/` folder mandatory.
- Source files `.ts` + compiled `.qm`.
- Zero hardcoded string in Views, Controllers, Models.

If disabled:
- No `self.tr()` call required (strings stay hardcoded in French in the Views).
- No `resources/i18n/` folder.

---

## Stack

- `PyQt6.QtCore.QTranslator` (native Qt, zero external dependency).
- `pylupdate6` (shipped with PyQt6) to extract strings → `.ts`.
- `lrelease` (Qt Linguist Tools — install separately if needed) to compile `.ts` → `.qm`.

---

## Mandatory structure

```
resources/
└── i18n/
    ├── app_fr.ts             # FR source (human translations)
    ├── app_en.ts             # EN source
    ├── app_fr.qm             # Compiled — loaded by the application
    └── app_en.qm             # Compiled
```

---

## Conventions

- Every user-visible string goes through `self.tr("text")` in the Views.
- No `self.tr` in Models / Controllers — business logic ignores the language.
- Exceptions raised by Models contain French (support language); the Controller translates the message on the View side via `self.tr(str(exception))` if needed.
- String format: avoid concatenations. Use `self.tr("Bonjour %1").arg(name)`.

---

## Initialization in main.py

```python
from PyQt6.QtCore import QTranslator, QLocale
from PyQt6.QtWidgets import QApplication
from pathlib import Path

import config


def install_translator(app: QApplication) -> QTranslator:
    """Installe le traducteur selon la préférence utilisateur."""
    locale = _load_user_locale()  # depuis preferences.json
    translator = QTranslator()
    qm_path = Path("resources/i18n") / f"app_{locale}.qm"
    if translator.load(str(qm_path)):
        app.installTranslator(translator)
    return translator
```

## Developer workflow

### 1. Extract strings from the code

```powershell
pylupdate6 (Get-ChildItem -Recurse views\*.py | Select-Object -ExpandProperty FullName) `
           -ts resources\i18n\app_fr.ts resources\i18n\app_en.ts
```

### 2. Translate (Qt Linguist)

Open each `.ts` in Qt Linguist, fill in the translations, save.

### 3. Compile to .qm

```powershell
lrelease resources\i18n\app_fr.ts resources\i18n\app_en.ts
```

`lrelease` is not bundled in PyQt6. Two options:
- Install Qt Linguist Tools separately (download.qt.io).
- Use `PyQt6.lrelease` if available in the installed PyQt6 version.

### 4. Automation script — `scripts/i18n_update.ps1`

Delivered in the last application batch if i18n is enabled:

```powershell
# scripts/i18n_update.ps1
$files = Get-ChildItem -Recurse -Include *.py -Path views,controllers `
         | Select-Object -ExpandProperty FullName
pylupdate6 $files -ts resources\i18n\app_fr.ts resources\i18n\app_en.ts
Write-Host "Chaînes extraites — éditer les .ts puis lancer i18n_compile.ps1"
```

And `scripts/i18n_compile.ps1`:

```powershell
Get-ChildItem resources\i18n\*.ts | ForEach-Object {
    lrelease $_.FullName
}
Write-Host "Fichiers .qm générés"
```

## Hot language change — not supported

- No dynamic `QTranslator` reinstallation on the fly.
- Language change = write to `preferences.json` + invite to restart.
- This constraint avoids the complexity of re-translating all open widgets.

---

## Anti-patterns — what NOT to do

- **Do not** hardcode a user string in a View (`QLabel("Texte")` → `QLabel(self.tr("Texte"))`).
- **Do not** put a UI string in a Model or a Controller.
- **Do not** hot-reload translations.

## Integrity verification

Detailed in `@rules/verification.md`. Key points (if i18n = Yes): `resources/i18n/app_{fr,en}.{ts,qm}` present; no French `QLabel("…")`/`QPushButton("…")` hardcoded in `views/`; `QTranslator` installed in `main.py` before the `MainWindow`; `language` preference read from `preferences.json` at startup.
